# 写了一个适合 Agent 生成 React 代码的编译器

我们用 spec 驱动开发，一份 spec 描述实体，让 agent 据此生成整套页面。一个标准 CRUD 实体要列表页、详情页、编辑页——几个 React 组件。问题是每来一个新实体就让 LLM 重画一遍：慢，而且 output token 很贵。

更难受的是这些页面之间高度重复。订单页和商品页都是「ProTable + 筛选 + 行操作」那套骨架，组件早就复用了，但页面本身的代码——列定义、请求、跳转、删除确认——结构几乎一样，只是字段不同。让 LLM 一遍遍把这套结构重新写出来，是在为重复付钱。

重复的活该交给模板。

## 别让 LLM 生成代码

第一反应可能是「那让 LLM 把模板也生成了」。不对。LLM 生成代码有三个毛病：不可复现（同样的输入两次输出不一样）、贵、没法校验（你不知道它这次有没有偷偷改了点什么）。

所以铁律立在这儿：**LLM 只产 model，渲染机械确定**。skill 文档第一行就是这句：

> 零 LLM 渲染：LLM 只产 model，模板机械渲染——可复现、可逐字节校验。

LLM 的活缩到只填一份 SDM（几十行 JSON：实体名、字段、路由前缀）。剩下的交给 Eta 模板机械渲染。同一份 SDM + 同一套模板，永远吐出字节相同的代码。

## 模板不是设计出来的，是反推出来的

这步是我觉得最有意思的。模板不是我坐下来凭空设计一套占位符，而是**从一个已经写好的真实页面反推**出来。

`templatize-init --from-scene <真实页面>` 先把那个页面的源码原样快照到 `__rendered__/` 当基线。然后读这份源码，反推出它对应的 SDM——只列模板会参数化的字段，不强求把源里每个字段都列进去。最后逐文件把 `.tsx` 转成 `.eta`：

```tsx
// 原页面里写死的
export default function BusinessPartnerPage() { ... }
// 转成模板，只挖类型名和身份字面值
export default function <%= it.name %>Page() { ... }
columns: [
<% it.fields.forEach(function (f) { %>  { title: '<%= f.label %>', dataIndex: '<%= f.name %>' },
<% }) %>]
```

关键决策是**绝大多数页面代码保持 literal，只参数化两类东西**：类型名（所有 `<Entity>` 泛型/接口）和身份字面值（'业务伙伴'、'/api/business-partners' 这种）。这条边界是 survey 了 47 个真实页面才定下的——参数化挖得越少，模板越稳；挖过头，模板就脆。

## 守门一：逐字节回放

模板从某个页面抽出来，怎么证明它没在抽取时改坏语义？

`templatize-verify`：用反推出的那份 SDM 去渲染刚做好的模板，结果必须和 `__rendered__/` 里的原页面**逐字节相等**（`bytesEqual`，长度和内容都一致）。一个字符对不上就报 diff。

这一步保证模板对它的「出身页面」无损——参数代回去，原封不动还原。

## 守门二：交叉验证

但无损还原自己的出身页面，不代表它能泛化到别的实体。模板很可能把某个只属于训练页面的东西硬编码了进去。

于是有第二道门，工具里叫 `crossfold`——本质是**交叉验证**。

做法：准备几个同形态、不同实体的页面的 SDM（标准 CRUD 这类，拿 `payment-term` 当训练抽模板，再拿另外几个实体当验证）。用验证实体的 SDM 去渲染同一套模板，把产物丢进临时目录跑 typecheck 和契约检查（关键 import、组件有没有出现）。验证实体都能过，说明模板是泛化的；炸了，说明哪里还粘着训练页面的私货。

两档严格度：`scaffolding` 只看语法错（TS1xxx，结构对不对），`strict` 连字段层类型（TS2xxx）都查。

这就是机器学习里的 **k-fold cross-validation**——一份当训练、其余当验证，轮着来；极端情形是 leave-one-out。（顺嘴：这个词是 cross-validation，不是 grid search，后者是调超参的，两码事。）搬到「模板抽取」上意外地贴：模板就是从一份数据里学出来的泛化器，理应拿留出来的数据验它有没有过拟合到训练样本。

## 到这儿

一份 SDM 进去，一套页面确定性地出来，可逐字节验、可交叉验证。LLM 只在最前面填那几十行 model，再不碰渲染。

但有个天花板我当时没绕过去：Eta 模板吐出来的终究是**字符串**。字符串没法被程序操纵——你没法 diff 它的结构、没法对它做变换、没法让另一个工具读懂它到底要画什么。模板里还混着行为（删除 handler、跳转）。

下一篇讲怎么掀掉这层天花板：让模板别再生成代码，改成生成数据。
